Datasets no longer auto-save on construction. Persistence is now always explicit via `save()`, and `Dataset.filepath` is optional so in-memory / region-scoped copies can exist without a disk destination. Region scoping previously built a filtered copy that reused its source file's path, and construction silently truncated that shared file — corrupting subsequent simulations that reused the same dataset file (e.g. state-by-state runs in a reused worker container). `run()` now loads the input dataset from disk only when the caller did not already supply in-memory data.
